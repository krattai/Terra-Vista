New SideBox Information

PHP-Nuke Addons & Modules System

Since PHP-Nuke 5.0 you can add new modules, addons or plugins simply copying the addons files into a directory. With this feature, PHP-Nuke gains modularity and you, the webmaster, the choice to install and/or unistall the modules you want with an easy step.

Now this new idea come out from Rob Sutton is to make it even modular. Current module system will force developer to edit mainfile.php especially if you want to add sidebox. With this new system user don't have to worried anymore. As long you used the themes come with the package. Woalla!

This document has two main parts, one for webmasters/users and other for Addons developers.

Please read it carefully and remember that this system, as the whole NukeAddOn comes without any warranty and all you do from here is under your own responsability and risk. Always remember to backup your database and all your files before doing anything.

1. Information for Webmasters/Users

The new sidebox system is now can be place easily without changing anything on your mainfile. This way it's really modular. This system invented by Rob Sutton and now bundle into our current AddOn.

In order to make the box appear correctly you must have replace all themes files include on NukeAddOn and then to add it you just need to create a new sidebox. However you must replace the comment in the contents field you need to place the exact URL where the modules script located. E,g. Calendar modules url are modules/Calendar/addonblock.php.
